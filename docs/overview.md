\# ExamVault



\## Overview



ExamVault is a unified assessment platform that enables educational institutions to conduct comprehensive examinations in a secure, monitored environment. It brings together multiple question types—MCQs, coding challenges, and language skill assessments—into a single cohesive platform with browser-based proctoring and hybrid grading.



\## Roles



\### Teachers

Educators who create and manage assessments. They can:

\- Build exams with diverse question types (MCQ, text, code, listening, reading, writing, speaking)

\- Upload media assets (audio files, reading passages)

\- Configure exam settings (duration, passing scores, access codes)

\- Review and grade subjective responses

\- Manage a reusable question bank with tagging and difficulty levels

\- Import saved questions directly into new exams

\- Track student performance and results



\### Students

Learners who take assessments. They can:

\- Join exams using access codes shared by teachers

\- Complete assessments in a proctored fullscreen environment

\- Receive immediate feedback on auto-graded portions

\- View detailed results after grading is complete



\## Key Scenarios



1\. \*\*Exam Creation\*\*: Teacher creates a new exam, adds sections (listening, reading, writing, etc.), uploads media, configures time limits and access codes



2\. \*\*Taking an Exam\*\*: Student enters access code, confirms camera/microphone access, enters fullscreen proctored mode, completes questions with timer visible



3\. \*\*Proctoring Enforcement\*\*: System monitors tab switches and fullscreen exits. After 3 violations, exam auto-submits. Warnings displayed in real-time



4\. \*\*Hybrid Grading\*\*: MCQs score immediately upon submission. Writing, speaking, and code responses queue for teacher review. Total score updates as grading completes



5\. \*\*Results Review\*\*: Student sees immediate auto-graded score, then receives final score once teacher completes manual review



\## Question Types



| Type | Content | Grading |

|------|---------|--------|

| MCQ | Multiple choice options | Auto-graded |

| Short Text | Brief text response | Auto (keyword) or manual |

| Code | Syntax-highlighted editor | Manual review |

| Listening | Audio + questions | Auto-graded |

| Reading | Passage + comprehension | Auto-graded |

| Writing | Essay prompt | Manual review |

| Speaking | Audio recording | Manual review |



\## Design Direction



\*\*Aesthetic\*\*: Clean, professional, and trust-inspiring. The interface should feel secure and serious—befitting a high-stakes assessment environment—while remaining intuitive and approachable.



\*\*Color Palette\*\*: Deep navy and slate tones for authority, accented with teal/cyan for interactive elements. Warning states use amber, success uses emerald.



\*\*Typography\*\*: Modern sans-serif with clear hierarchy. Bold headings for navigation confidence, comfortable body text for extended reading during exams.



\*\*Motion\*\*: Subtle, purposeful animations. Smooth transitions between exam sections, gentle pulse on timer when time is low, clear visual feedback on answer selection.
